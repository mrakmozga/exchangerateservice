# CNB Exchange Rate Service

Сервис для синхронизации и анализа курсов валют [Чешского национального банка (ČNB)](https://www.cnb.cz). Загружает данные по расписанию, хранит в SQLite и предоставляет Web API + веб-интерфейс для отчётов.

---

## Возможности

- **Автоматическая синхронизация** курсов по cron-расписанию (настраивается через UI)
- **Синхронизация за период** — за день, диапазон дат или полный год
- **Отчёт по периоду** — минимальный, максимальный и средний курс для выбранных валют
- **Веб-интерфейс** — управление синхронизацией и просмотр отчётов прямо в браузере
- **REST API** с автоматической документацией (Swagger UI)
- **Docker-деплой** — одна команда для запуска

### Описание модулей

#### `main.py`
Точка входа. Отвечает за:
- Инициализацию FastAPI-приложения и SQLite при старте (`lifespan`)
- Регистрацию и перепланирование задачи в APScheduler
- Все HTTP-эндпоинты: синхронизация, расписание, отчёт, статус
- Раздачу статического веб-интерфейса

#### `cnb_client.py`
Общение с API ČNB. Отвечает за:
- HTTP-запросы к двум эндпоинтам ČNB (`daily.txt`, `year.txt`)
- Парсинг обоих форматов данных (дневной и годовой)
- Оптимизацию запросов: полные годы → один запрос `/year.txt`, частичные периоды → итерация по `/daily.txt`
- Обработку аномалий: пустые значения, `N/A`, HTTP-ошибки

#### `sync_service.py`
Тонкий оркестрирующий слой. Отвечает за:
- Вызов `cnb_client` для получения данных
- Передачу результата в `database` для сохранения
- Логирование хода синхронизации

#### `database.py`
Персистентное хранилище. Отвечает за:
- Инициализацию схемы БД (`CREATE TABLE IF NOT EXISTS`)
- UPSERT-запись курсов (`ON CONFLICT DO UPDATE`) — защита от дублей
- Агрегатный запрос для отчёта (`MIN`, `MAX`, `AVG` с нормализацией `rate / amount`)
- Автоматическое создание директории для файла БД

#### `config.yaml`
Единственный файл конфигурации. Содержит:
- Cron-расписание и часовой пояс планировщика
- Список валют для синхронизации
- Путь к файлу SQLite
- URL-адреса API ČNB

#### `static/index.html`
Веб-интерфейс (один HTML-файл без фреймворков). Содержит:
- Форму запуска синхронизации за сегодня с отображением статуса последнего запуска
- Форму синхронизации за произвольный период
- Настройку расписания (время + часовой пояс) с сохранением на сервере
- Форму отчёта с таблицей мин/макс/среднее по валютам

## Источник данных

Данные берутся напрямую с сайта ČNB в виде открытых текстовых файлов:

```
# Курс за конкретный день
https://www.cnb.cz/.../daily.txt?date=27.07.2019

# Все курсы за год
https://www.cnb.cz/.../year.txt?year=2019
```

Формат ответа — таблица с разделителем `|`. Поле `Amount` указывает, за сколько единиц валюты дан курс (для JPY, HUF — 100). В отчёте все значения нормализуются к 1 единице: `rate / amount`.

## API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/` | Веб-интерфейс |
| `GET` | `/health` | Проверка работоспособности |
| `GET` | `/sync/status` | Время последней синхронизации и текущее расписание |
| `POST` | `/sync/daily` | Запустить синхронизацию за сегодня |
| `POST` | `/sync/range` | Синхронизация за период (`start_date`, `end_date`) |
| `POST` | `/sync/schedule` | Обновить расписание (`hour`, `minute`, `timezone`) |
| `GET` | `/report` | Отчёт по периоду (`start_date`, `end_date`, `currencies[]`) |
| `GET` | `/docs` | Swagger UI |

### Пример: получить отчёт

```bash
curl "http://localhost:8000/report?start_date=2024-01-01&end_date=2024-03-31&currencies=USD&currencies=EUR"
```

```json

  "start_date": "2024-01-01",
  "end_date": "2024-03-31",
  "currencies": ["USD", "EUR"],
  "results": [
    {"currency": "EUR", "min_rate": 24.51, "max_rate": 25.41, "avg_rate": 24.98, "data_points": 65},
    {"currency": "USD", "min_rate": 22.14, "max_rate": 23.87, "avg_rate": 22.95, "data_points": 65}
  ],
  "missing": []
}
```

## Конфигурация

Файл `config.yaml` монтируется с хоста — изменения применяются без пересборки образа:

```yaml
scheduler:
  cron: "1 0 * * *"      # каждый день в 00:01

timezone: "Europe/Moscow" # часовой пояс расписания

currencies:               # валюты для синхронизации
  - USD
  - EUR
  - GBP
  - JPY

database:
  path: "data/cnb_rates.db"
```

Часовой пояс и расписание также можно изменить через веб-интерфейс — изменения сразу сохраняются в `config.yaml` и применяются без перезапуска контейнера.

---

## Хранение данных

SQLite-файл хранится в именованном Docker volume `cnb_data` — данные сохраняются между перезапусками контейнера.

```bash
# Посмотреть данные
docker exec cnb_exchange_service sqlite3 data/cnb_rates.db "SELECT * FROM exchange_rates LIMIT 5;"

# Удалить все данные
docker compose down -v
```
